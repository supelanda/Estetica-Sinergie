Setup Supabase — istruzioni SQL (solo sviluppatore)

Esegui questo SQL nel pannello Supabase → SQL Editor per creare la tabella e configurare Row Level Security. Poi inserisci le tue credenziali nelle 4 costanti in cima allo script di questa pagina. 

-- ═══════════════════════════════════════════════════════
-- Tabella domande — richieste delle clienti dal sito
-- ═══════════════════════════════════════════════════════
create table domande (
  id          uuid default gen_random_uuid() primary key,
  created_at  timestamptz default now(),
  nome        text not null,
  telefono    text,
  email       text,
  servizio    text,
  messaggio   text not null,
  stato       text default 'nuova'
              check (stato in ('nuova','letta','risposta')),
  risposta    text,
  risposta_at timestamptz
);

-- Abilita Row Level Security
alter table domande enable row level security;

-- Chiunque può inserire dal form pubblico (usa anon key nel form)
create policy "insert_public" on domande
  for insert to anon with check (true);

-- Solo utenti autenticati (Barbara) possono leggere e modificare
create policy "all_authenticated" on domande
  for all to authenticated using (true);

-- Indice per ordinamento per data
create index idx_domande_created on domande(created_at desc);

-- ═══════════════════════════════════════════════════════
-- Abilita Realtime (notifiche nuove domande senza refresh)
-- ═══════════════════════════════════════════════════════
-- Supabase → Database → Replication → abilita "domande"

-- ═══════════════════════════════════════════════════════
-- Utente admin: Supabase → Authentication → Users → Add User
-- Inserisci email e password con cui Barbara accederà a questa pagina.
-- ═══════════════════════════════════════════════════════

Per la pagina admin usa la service_role key (non anon key) — può bypassare RLS. Trovala in Supabase → Project Settings → API → service_role.
